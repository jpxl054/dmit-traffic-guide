## DMIT 流量超了到底会发生什么？

先说结论：**不同套餐，处理方式不一样。**

这是很多用户搞混的地方。DMIT 并不是一刀切地「超了就停机」，而是根据你买的套餐系列，执行不同的超量策略。

### 策略一：降速不停机（Soft Suspend）

这是目前 DMIT 主流的处理方式，适用于大部分 EB 系列和 T1 系列套餐。

当你的月度流量用完后，服务器**不会关机**，而是自动把带宽限速到一个非常低的水平（通常在 1Mbps～10Mbps 之间，具体数值视套餐而定）。你的机器还在线，SSH 能连，网站能访问，只是速度慢得像拨号时代。

对于很多场景来说，这其实挺人性化的：至少不会半夜突然断网，你还有机会登进去处理紧急事务、迁移数据，或者充值购买额外流量包。

### 策略二：直接暂停服务（Hard Suspend）

这种情况主要出现在部分 Pro 系列套餐上，尤其是采用**双向流量计费**（IN + OUT 合并计算）的套餐。

一旦流量用尽，服务器会直接暂停，无法访问，直到下个计费周期重置，或者你手动购买流量包续命。

所以在购买 DMIT 套餐之前，**弄清楚你买的套餐用的是哪种超量策略，比什么都重要。**

## 搞懂流量计费规则，才能真正用好 DMIT

为什么同样是「1TB 流量」，有些套餐用得快，有些用得慢？这跟计费方式直接相关。

### 单向计费（MAX 模式）

DMIT 的 T1 系列和部分高端套餐采用的是 `MAX(IN, OUT)` 计费方式——**只取进出流量中较大的那个值**，而不是两者相加。

举个例子：你这个月下载了 500GB，上传了 200GB，那实际计费只有 500GB，不是 700GB。

这对于**以下载为主**的使用场景（比如科学上网、视频流量）来说非常友好，流量利用率高出不少。

### 双向计费（IN + OUT 合并）

部分 Pro 系列套餐把上行和下行流量**全部加起来**计算。同样是下载 500GB + 上传 200GB，计费就是 700GB。

如果你的业务有大量上行流量（比如备份、文件传输、视频直播推流），双向计费会让你的流量消耗速度翻倍，需要特别注意。

## 流量超了之后，有哪些解决方案？

好，流量超了，现在怎么办？有几条路可以走。

### 方案一：等待下个月自动重置

最省钱，但代价是这个月剩余时间只能用龟速（如果是降速套餐）或者直接断服（如果是暂停套餐）。

适合：流量刚超一点点，而且这个月快结束了，可以凑合几天。

### 方案二：购买额外流量包

DMIT 支持在控制面板购买额外的流量包，直接给当前套餐补充流量，立即生效，机器恢复正常速度。

这是最快的解决方案，适合**流量超出较多、还有大量时间到月底**的情况。

操作路径：登录 DMIT 用户后台 → 进入对应服务 → 找到「Add-ons」或流量包购买选项 → 完成支付即可。

### 方案三：升级到更高流量套餐

如果你发现每个月都在超量，那说明当前套餐的流量根本不够用，与其反复买流量包，不如直接升级套餐。

DMIT 的套餐层级很丰富，从入门的几十 GB 到几十 TB 都有，一步步往上升就好。

### 方案四：手动切换超量策略

这是很多人不知道的一个功能：**DMIT 允许用户在控制面板手动切换超量策略**——是降速还是停机，你可以自己选。

如果你更在意服务不中断，就保持降速模式；如果你不想欠费、希望超量后直接停机等重置，也可以切换回去。

进入 DMIT 后台，在对应服务的管理页面里，找到「Bandwidth Overage Policy」相关选项，根据自己需求切换即可。

## 如何提前防止流量超量？

与其事后补救，不如提前预防。

### 1. 开启流量用量告警

DMIT 后台提供流量使用量监控功能，你可以设置在达到一定百分比时发送邮件通知。建议把告警阈值设在 **70%** 和 **90%** 两档，留足反应时间。

### 2. 在 VPS 上部署流量监控工具

除了 DMIT 自带的后台监控，也可以在 VPS 上安装开源监控工具，比如 `vnStat`，实时记录每日、每月的流量消耗明细，方便排查哪个进程在偷跑流量。

bash
# 安装 vnStat（Debian/Ubuntu）
apt install vnstat -y
# 查看月度流量统计
vnstat -m


### 3. 选对套餐系列

根据你的实际用途选线路和套餐，不要为了「线路好」而选一个流量配额严重不够用的套餐，反而得不偿失。

- **科学上网/代理，用量大**：优先考虑 LAX.EB 或 SJC.T1，流量更充裕，性价比高
- **中国大陆用户，对延迟敏感**：LAX.Pro 或 HKG.Pro，线路质量有保障
- **需要 DDoS 防护**：SJC.T1 系列，带 20Gbps 防护

### 4. 善用单向计费套餐

如果你的业务以下载流量为主（下行 >> 上行），优先选择支持 `MAX(IN, OUT)` 计费模式的套餐，相当于变相给你的流量打了个折扣。
